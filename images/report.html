데이터 수집 및 전처리

📌 전처리 단계

  • HTML 태그 제거 <.*?>
  • URL 제거 http[s]?://
  • 멘션 제거 @username
  • 한글·영문·숫자 외 문자 제거
  • 연속 공백 정규화
  • 한국어 포함 댓글만 유지

🔤 형태소 분석 (Okt)

  • 추출 품사: Noun Adjective Verb
  • 정규화(norm=True) + 어간 추출(stem=True)
  • 2글자 이상 토큰만 사용
  • 불용어 18개 제거
  • (진짜·너무·정말·댓글·영상 등)

💡 라벨링 전략: 자동 확장 우선 사용

auto_full_results/comments_labeled_auto_full.csv가 존재하면 우선 사용하고, 없으면 수동 라벨 파일(comments_labeled.csv)을 사용합니다. 이를 통해 소규모 수동 라벨에서 대규모 학습 데이터로 확장할 수 있습니다.

딥러닝 모델 구조

공통 하이퍼파라미터

Vocab Size: 5,000 Embed Dim: 64 Learning Rate: 1e-3 Batch Size: 32 Max Epochs: 12 Optimizer: Adam Loss: Binary Crossentropy EarlyStopping: patience=3

🔷 CNN 모델

Embedding(vocab, 64, input_length)
Conv1D(128, kernel=5, relu)
GlobalMaxPooling1D()
Dense(64, relu)
Dropout(0.3)
Dense(1, sigmoid)

Conv1D로 지역적 n-gram 패턴을 포착. 학습 속도가 빠르며 단기 패턴에 강점.

🔶 LSTM 모델

Embedding(vocab, 64, input_length)
LSTM(64)
Dense(32, relu)
Dropout(0.3)
Dense(1, sigmoid)

순방향 시퀀스 의존성 학습. 장거리 문맥 파악에 유리한 순환 구조.

🔺 Bi-LSTM 모델

Embedding(vocab, 64, input_length)
Bidirectional(LSTM(64)) # 순방향 + 역방향 = 128dim
Dense(32, relu)
Dropout(0.3)
Dense(1, sigmoid)

양방향 문맥을 동시에 학습해 문장 전체의 의미를 더 풍부하게 포착. 세 모델 중 가장 많은 파라미터를 사용.
